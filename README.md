# Cover Pro 使用手册

Cover Pro 是一个通用封面制作工具，用来快速制作小红书、抖音、视频号、微信公众号等平台的封面图。它提供多种模板、尺寸预设、文字排版、图片上传、元素拖拽、主题色和 PNG 导出能力。

![Cover Pro 工作台](docs/screenshots/01-workspace.png)

## 功能概览

- 多平台尺寸：通用 3:4、小红书竖版/横版、抖音竖版/横版、视频号竖版/横版、微信公众号封面。
- 多种模板：赛博霓虹、终端信息流、雷达聚焦、玻璃简报、发布卡片、社媒贴纸、清单笔记、深度头图、醒目数字、时尚杂志。
- 内容编辑：标签、主标题、副标题、图形文字、账号、徽章都可以手动输入。
- 视觉控制：模板、主题色、自定义背景色、每个文本块颜色、字体、字号、行高、间距、宽度。
- 元素位置：支持在画布上拖拽文字元素，也支持在面板里精确调整 X/Y。
- 图片素材：支持上传、替换、删除图片，并调整图片大小和位置。
- 显示控制：可单独显示/隐藏标签、副标题、账号、徽章、图形文字、图片、安全区。
- 保存导出：自动保存草稿，支持 `Ctrl + S` 手动保存，支持下载 PNG。

## 快速开始

### 1. 安装依赖

```bash
npm install
```

### 2. 启动开发服务

```bash
npm run dev
```

默认访问地址：

```text
http://localhost:5173
```

如果 5173 端口被占用，Vite 会自动使用其他端口，请以终端输出为准。

### 3. 构建生产版本

```bash
npm run build
```

构建产物会输出到 `dist/`。

### 4. 本地预览生产版本

```bash
npm run preview
```

## 部署到 GitHub Pages

项目已经内置 GitHub Actions 部署配置：

```text
.github/workflows/deploy-pages.yml
```

推送到 GitHub 的 `main` 分支后，Actions 会自动安装依赖、构建项目，并把 `dist/` 发布到 GitHub Pages。

### 第一步：创建 GitHub 仓库

在 GitHub 上创建一个仓库，例如：

```text
cover-pro
```

### 第二步：初始化本地 Git 仓库

如果当前目录还不是 Git 仓库，可以执行：

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/<你的用户名>/cover-pro.git
git push -u origin main
```

把 `<你的用户名>` 替换成你的 GitHub 用户名。如果仓库名不是 `cover-pro`，也要同步替换远程地址里的仓库名。

### 第三步：开启 GitHub Pages

进入 GitHub 仓库页面：

```text
Settings -> Pages
```

在 Build and deployment 中选择：

```text
Source: GitHub Actions
```

之后每次推送 `main` 分支，GitHub Actions 都会自动发布。

### 第四步：访问线上地址

部署完成后，访问地址通常是：

```text
https://<你的用户名>.github.io/cover-pro/
```

如果仓库名不是 `cover-pro`，地址最后一段会变成你的仓库名。

### 关于 Vite base 路径

GitHub Pages 的项目站点通常部署在子路径下，例如 `/cover-pro/`。项目已经在 `vite.config.ts` 中配置：

```ts
base: process.env.VITE_BASE_PATH ?? '/',
```

GitHub Actions 构建时会自动注入：

```text
VITE_BASE_PATH=/${{ github.event.repository.name }}/
```

所以本地开发仍然使用 `/`，线上会自动使用 `/<仓库名>/`，不需要手动修改。

## 基本使用流程

### 第一步：选择平台尺寸

左侧「平台尺寸」区域提供常用封面尺寸。点击任意尺寸后，中间画布会按该尺寸比例展示。

当前内置尺寸：

| 用途 | 尺寸 |
| --- | --- |
| 通用封面 3:4 | 1080 x 1440 |
| 小红书视频封面竖版 | 1080 x 1440 |
| 小红书视频封面横版 | 1440 x 1080 |
| 抖音视频封面竖版 3:4 | 1242 x 1660 |
| 抖音视频封面竖版 9:16 | 1080 x 1920 |
| 抖音视频封面横版 | 1080 x 608 |
| 视频号视频封面竖版 | 1080 x 1260 |
| 视频号视频封面横版 | 1080 x 608 |
| 微信公众号封面 | 900 x 383 |

### 第二步：输入封面内容

在左侧「封面内容」中填写：

- 标签：用于小标题、栏目名、主题分类。
- 主标题：封面的核心大标题。
- 副标题：补充说明、卖点、摘要、清单内容。
- 图形文字：会显示在模板里的视觉图形块上。
- 账号：作者、账号名、品牌名。
- 徽章：重点推荐、编号、卖点标签等。

文字输入框支持换行，画布中的对应文本也会跟随换行。

### 第三步：选择模板

右侧「模板」区域可以切换封面风格。

![模板与配置面板](docs/screenshots/02-settings.png)

模板适用建议：

| 模板 | 适合内容 |
| --- | --- |
| 赛博霓虹 | 科技感、强视觉冲击、趋势、工具、观点 |
| 终端信息流 | 教程、方法论、效率、知识内容 |
| 雷达聚焦 | 趋势、观察、清单、重点发现 |
| 玻璃简报 | 正式内容、品牌内容、信息型封面 |
| 发布卡片 | 新品、活动、课程、内容推荐 |
| 社媒贴纸 | 轻量、活泼、推荐类封面 |
| 清单笔记 | 集合、步骤、排行、多点总结 |
| 深度头图 | 长文、专题、公众号头图 |
| 醒目数字 | 排行、清单、结果、强卖点 |
| 时尚杂志 | 品牌、人物、生活方式、高质感内容 |

### 第四步：设置主题色

右侧「主题色」提供内置配色，也可以使用自定义颜色。

可调内容包括：

- 主题色。
- 背景色。
- 每个文本块的字体颜色。
- 页面系统主题：右上角支持浅色/暗黑界面切换。

### 第五步：上传图片

左侧「图片素材」可以上传 Logo、截图、产品图或其他封面素材。

上传后可以：

- 替换图片。
- 删除图片。
- 调整图片大小。
- 调整图片水平/垂直位置。

注意：上传图片和图形文字是两个独立元素。图片控制只影响上传的图片，图形文字的位置通过「元素位置」或画布拖拽控制。

### 第六步：调整字体和排版

右侧「排版」中可以调整：

- 主标题字号。
- 主标题行高。
- 主标题字距。
- 主标题字重。
- 主标题宽度。
- 副标题字号。
- 副标题行高。
- 副标题上间距。
- 副标题宽度。
- 背景亮度。

字体支持系统字体、衬线、等宽、展示字体、圆体、窄体、手写、书法风格，也支持上传自定义字体。

### 第七步：拖拽和精确定位元素

画布中的标签、主标题、副标题、账号、徽章、图形文字可以直接拖动。

也可以在「元素位置」里选择某个元素，然后用 X/Y 滑杆做精确调整。

可定位元素：

- 标签。
- 主标题。
- 副标题。
- 账号。
- 徽章。
- 图形文字。

拖拽时元素不会完整移出画布，方便做边缘构图，同时避免导出时找不到元素。

### 第八步：控制显示内容

在「显示内容」中可以单独开关：

- 标签。
- 副标题。
- 账号。
- 徽章。
- 图形文字。
- 图片。
- 安全区。

安全区用于辅助检查重要文字是否在平台裁切范围内。最终导出前可以隐藏安全区。

### 第九步：下载 PNG

确认画面后，点击右上角「下载 PNG」即可导出当前封面。

![画布预览](docs/screenshots/03-canvas.png)

导出的图片会按当前选择的平台尺寸生成，例如 1080 x 1440、1440 x 1080、900 x 383 等。

## 保存机制

Cover Pro 有两种保存概念：

### 草稿保存

日常编辑内容会自动保存到浏览器本地存储。刷新页面后会尽量恢复上一次的设计。

你也可以按：

```text
Ctrl + S
```

手动保存当前设计。保存成功后页面顶部会出现绿色提示。

### PNG 导出

草稿保存只保存编辑状态，不会生成图片文件。要获得最终封面图片，请点击「下载 PNG」。

## 面板说明

### 左侧面板

左侧主要负责内容和素材：

- 平台尺寸：选择输出尺寸。
- 封面内容：填写文字内容。
- 图片素材：上传、替换、删除图片。

### 中间画布

中间是实时预览区：

- 所见即所得展示封面。
- 支持拖拽文字元素。
- 显示当前画布尺寸。
- 可显示/隐藏安全区。

### 右侧面板

右侧主要负责视觉样式：

- 模板：切换整体风格。
- 主题色：切换或自定义配色。
- 文字颜色：分别设置标签、主标题、副标题、账号、徽章、图形文字颜色。
- 排版：控制字号、行高、字距、宽度等。
- 元素位置：精确控制元素坐标。
- 显示内容：控制组件显隐。
- 装饰与图片：控制装饰密度、图片大小和图片位置。

## 修改模板默认布局

当前项目已经把调试保存后的布局写入 `src/styles/template-layouts.css`，并作为默认布局使用。

默认情况下，页面不会显示「调试：写入当前模板默认布局」按钮。

如果后续需要继续调试模板默认位置，可以打开配置开关：

```ts
// src/config/features.ts
export const coverToolFeatures = {
  enableTemplateLayoutDebug: true,
} as const;
```

然后启动开发服务：

```bash
npm run dev
```

操作流程：

1. 在页面选择目标平台尺寸和目标模板。
2. 拖动画布上的元素，或在「元素位置」中调整 X/Y。
3. 点击「调试：写入当前模板默认布局」。
4. 对应默认布局会写入 `src/styles/template-layouts.css`。
5. 调整完成后，把 `enableTemplateLayoutDebug` 改回 `false`。

注意：这个调试功能依赖 Vite 开发服务中的本地写入接口，只适合开发环境使用，不建议在生产环境开启。

## 自定义模板

模板文件位于：

```text
src/templates/
```

模板元信息位于：

```text
src/data/templates.ts
```

模板样式主要位于：

```text
src/styles/globals.css
src/styles/template-layouts.css
```

建议：

- 新增模板时，先在 `src/types.ts` 中补充 `TemplateId`。
- 再在 `src/data/templates.ts` 中添加模板名称、描述和默认主题。
- 新建对应的模板组件。
- 在 `src/components/CoverCanvas.tsx` 中挂载模板。
- 默认布局写在 `src/styles/template-layouts.css` 中。

## 自定义尺寸

平台尺寸配置位于：

```text
src/data/platforms.ts
```

新增尺寸示例：

```ts
{
  id: 'custom-cover',
  name: '自定义封面',
  shortName: '自定义',
  group: 'General',
  width: 1080,
  height: 1350,
}
```

新增后，左侧「平台尺寸」会自动出现该选项。

## 自定义主题

内置主题位于：

```text
src/data/themes.ts
```

主题包含：

- `accent`：主强调色。
- `accent2`：第二强调色。
- `background`：背景色。
- `foreground`：主文字色。
- `muted`：弱文字色。

页面中也可以直接使用自定义主题色和背景色。

## 常见问题

### 下载 PNG 是空白怎么办？

先确认页面中的封面画布已经渲染完成，再点击「下载 PNG」。如果仍然异常，请重新启动开发服务后再试：

```bash
npm run dev
```

### 为什么保存后没有生成图片？

`Ctrl + S` 和自动保存保存的是编辑草稿，不会生成图片文件。生成图片需要点击「下载 PNG」。

### 为什么我看不到调试写入按钮？

默认关闭。需要在 `src/config/features.ts` 中把 `enableTemplateLayoutDebug` 改为 `true`。

### 调试写入按钮适合给普通用户使用吗？

不适合。它会修改项目里的模板默认 CSS，主要给开发者调模板布局使用。

### 修改模板默认位置后不生效怎么办？

确认修改的是 `src/styles/template-layouts.css`，并重新启动或刷新开发服务。该文件在 `src/main.tsx` 中晚于 `globals.css` 引入，因此它会覆盖基础样式。
